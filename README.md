# Spam-Email-Classification
This project implements a Spam Email Classification system using Machine Learning techniques. The workflow begins with data preprocessing using Pandas and NumPy, where email text is cleaned and structured for analysis. The textual data is then transformed into numerical feature vectors using TF-IDF (Term Frequency–Inverse Document Frequency) Vectorization via TfidfVectorizer, which assigns importance scores to words based on their frequency and relevance across the dataset. The dataset is split into training and testing sets using train_test_split to ensure unbiased evaluation. A Logistic Regression model is trained on these features to learn patterns that distinguish spam from legitimate (ham) emails.

In terms of working, when a new email is input, it first undergoes the same preprocessing and TF-IDF transformation. The trained Logistic Regression model then computes a weighted sum of the input features, where each word contributes differently based on learned coefficients. These weights reflect how strongly a word indicates spam or ham (e.g., words like “free”, “win”, “offer” may have higher spam weights). The result is passed through a sigmoid function to produce a probability score between 0 and 1. If the probability exceeds a defined threshold (typically 0.5), the email is classified as spam; otherwise, it is classified as ham. This probabilistic approach enables the model to make efficient and scalable binary classification decisions.
